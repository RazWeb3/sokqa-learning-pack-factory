## Quality issues found (no fix applied in this phase)

### Issue A-1: `learningStyle` only affects "audio"

| Setting | Implemented values | Effective |
|---|---|---|
| `learningStyle` | `audio`, `reading`, `quiz`, `balanced` | Only `audio` changes output |

`reading`, `quiz`, `balanced` produce **identical** document/quiz content.
The README lists all 4 as if they had distinct effects, which is misleading.

Evidence: `isAudioOptimized = profile.audioOptimization || profile.learningStyle === "audio"`.
No branch handles `reading`, `quiz`, or `balanced` separately.

**Severity: Medium.** Documented behavior does not match implementation.

### Issue A-2: `detailLevel` interacts poorly with `none`/`source_only`

`detailLevel: "detailed"` adds a "Deep Dive" section, but only in `none` mode
(`buildGenericDocumentItems`). In `source_only` and `source_plus` it is ignored.
Users who set `detailLevel: "detailed"` on a reference-based pack will see no change.

**Severity: Medium.** Behavior is mode-dependent but undocumented.

### Issue A-3: `targetUser`/`tone`/`difficulty` only affect framing, not content depth

These adjust labels, prefixes, and one choice wording — they do **not** change
vocabulary level, explanation length, or quiz difficulty substantively.
For example `difficulty: "hard"` only changes `choiceA` text; the distractors
(`choiceB/C/D`: "Ignore customer context" etc.) stay fixed and off-topic for
non-customer-service themes.

**Severity: High.** This is the biggest gap between marketing and reality.

### Issue A-4: `exampleLevel` examples are placeholders

`Example 1: Apply ${theme} to a practical scenario.` is a template string,
not a real example. `exampleLevel: "many"` adds 3 identical placeholder rows.

**Severity: Medium.** Placeholder content has low educational value.

### Issue A-5: `audioOptimization` shortening is coarse

`shortenForAudio` cuts at the first `.` / `:` / `。` / `：`. This can truncate
meaning ("Intro (Friendly): Let's learn" → "Intro (Friendly)" — drops the point).

**Severity: Low.** Functional but crude.

### Issue A-6: Fixed off-topic distractors

`choiceB/C/D` ("Ignore customer context", "Skip confirmation steps",
"Use unclear language") are hardcoded and only fit customer-service themes.
The 47-prefectures / solar-system packs in earlier investigation showed this.

**Severity: High.** Cross-theme quality blocker.

## README vs implementation consistency

| README claim | Matches code? |
|---|---|
| "All profile settings are optional. When omitted, defaults are applied." | Yes (`DEFAULT_PROFILE`) |
| `targetUser` → "Vocabulary, explanation depth" | Partial — only affects labels/prefix, not depth |
| `difficulty` → "Basics vs applied content" | Partial — only `choiceA` wording |
| `learningStyle` → "Short sentences vs explanations" | Partial — only `audio` works |
| `outputMode` → "Study / corporate training / exam prep" | Yes |
| `tone` → "Wording and register" | Yes |
| `detailLevel` → "Level of detail" | Partial — mode-dependent |
| `exampleLevel` → "Number of concrete examples" | No — placeholders, not concrete |
| `audioOptimization` → "SokQA-friendly short sentences" | Yes (coarse) |

**Recommendation for the v0.7 phase:** soften the README claims to match
reality, OR add real wiring in v0.7. This report informs that decision.

## Keep (essential reference / mode coverage)

- **4** `customer-service-exact-text` — only `exact_text_document` example
- **6** `multiple-references` — only multi-format example; also the CI test target

## Consolidate (group by intent)

- **2 + 3** (`source-only` + `source-plus`): same reference file, differ only in `mode`.
  Could become one config with `mode` switched by a preset/flag. Keep both for now
  since they are distinct *modes*, but note they can be preset-driven in v0.7.
- **7 + 8 + 9 + 10** (the 4 profile samples): same `none` mode, differ only in profile.
  Prime candidates for **v0.7 Preset Profiles** — collapse to one base config + a
  `preset` field (`beginner-audio`, `employee-training`, etc.). Keep all four until
  the preset system lands.
- **5** `multiple-docs-quizzes`: tests `documentCount`/`quizCount`. Orthogonal to the
  others. Keep, but it overlaps conceptually with any of 1–4 plus count overrides.

# 3. Future Quality-Improvement Candidates

These are **proposals only** — not implemented in v0.7's analysis phase.
They target the real quality gaps surfaced by the audit above.

## Candidates

| Candidate | What it controls | Fixes audit issue |
|---|---|---|
| `sentenceLength` | Target sentence length (chars / mora) for document & quiz TTS | A-5 (coarse audio shortening) |
| `explanationDepth` | Depth of quiz `explanation` (one-line / paragraph / multi-step) | A-2, A-3 (depth) |
| `quizStyle` | Quiz format: `concept-check` / `application` / `case-study` | A-3, A-6 (fixed distractors) |
| `practicalExamples` | Whether examples are real (reference-derived) or placeholder | A-4 (placeholder examples) |
| `vocabularyLevel` | Reading level / terminology ceiling | A-3 (targetUser depth) |
| `reviewQuestions` | Add a review/summary quiz at the end | New — reinforcement |
| `distractorSource` | Where wrong choices come from: `reference` / `theme` / `fixed` | A-6 (off-topic distractors) |
| `glossaryMode` | Build a key-term glossary section in the document | New — study aid |

## Prioritization recommendation

If v0.7 implements any of these, prioritize by impact:

1. **`quizStyle` + `distractorSource`** — directly fixes the worst current problem
   (fixed off-topic distractors, A-6). Highest user-visible impact.
2. **`explanationDepth`** — makes `difficulty` and `targetUser` meaningful (A-3).
3. **`practicalExamples`** — replaces placeholders with reference-derived examples (A-4).
4. **`sentenceLength`** — replaces the coarse `shortenForAudio` heuristic (A-5).
