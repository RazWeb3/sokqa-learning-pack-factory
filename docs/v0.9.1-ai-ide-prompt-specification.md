# v0.9.1 AI IDE Prompt Specification

> Status: Draft
> Scope: Prompt specification for AI IDE generation agents (Trae, Cursor, Codex, Gemini CLI, etc.)
> No implementation вҖ” specification only

---

## 0. Purpose

The Factory does not contain any teaching material data. All teaching materials are generated by AI IDEs using LLMs. This document defines the Prompt Specification that serves as the generation rules.

**No code changes allowed.** This is a design document only.

---


### Input Validation

Before LLM execution, the following are validated. If any check fails, **Generation Failed** without calling the LLM.

- `theme` вҖ” present and non-empty
- `language` вҖ” valid language code
- `profile` вҖ” one of: `beginner`, `student`, `employee`, `manager`
- `referenceMode` вҖ” one of: `none`, `source_plus`, `source_only`, `exact_text_document`
- `documentCount` вҖ” positive integer
- `quizCount` вҖ” non-negative integer

- Fallback templates are **prohibited**.
- Generation failure ends as **Generation Failed**.

- **е®ҹдҫӢгӮ’еҗ«гӮҒгӮӢ** вҖ” Include theme-specific, realistic examples

- **е…Ёе•Ҹз•°гҒӘгӮӢ** вҖ” Every question must test a unique viewpoint

## 5. Reference Mode

| Mode | Document | Quiz | Metadata |
|---|---|---|---|
| `none` | Generate from theme/title/description using general knowledge; keep educational | Generate from theme; domain-specific questions | `source.hasReference: false`, `source.mode: "none"` |
| `source_only` | Use only reference facts; no outside facts; reorganize only when meaning preserved | Derive questions from reference text only; no external knowledge | `source.hasReference: true`, `source.mode: "source_only"`, `source.path` populated |
| `source_plus` | Center reference facts; add learner-friendly explanation, examples, context | Use reference as base; generate domain-specific questions with reference-grounded distractors | `source.hasReference: true`, `source.mode: "source_plus"`, `source.path` populated |
| `exact_text_document` | Preserve original text exactly; no summarization, paraphrasing, or additions | **No quiz generated** (quizCount treated as 0) | `source.hasReference: true`, `source.mode: "exact_text_document"`, `quizzes: []` |

## 6. Retry Policy

- Based on `quality-report.json` results
- Maximum **2 retries** allowed
- Retry targets:
  - Document only
  - Quiz only
  - Both
- If still fails after retries вҶ’ **Generation Failed**
- Template generation is **prohibited** on retry

## 7.5. Prompt Quality Targets

These are the quality goals the **AI IDE should target** during generation вҖ” not validation checks.

| Target | Goal |
|---|---|
| Document similarity | < 20% |
| Quiz similarity | < 10% |
| Generic distractors | = 0 |
| Generic explanations | = 0 |
| Theme alignment | High |
| Practical examples | Required |

These targets guide the LLM generation behavior. Actual pass/fail is determined by **Validation (Section 7)**.
