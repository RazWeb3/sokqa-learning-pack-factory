# v0.9.0 Generation Quality Baseline

> Evaluated: 2026-06-30
> Baseline target: current repository behavior recorded as the v0.8.0 baseline for v0.9 comparison
> Evaluator: Factory `quality-report.json` + manual review
> Scope: investigation and evaluation only

## 1. Purpose

v0.9 では生成品質を継続改善する。改善前後を比較できるよう、現時点の生成品質をベースラインとして固定する。

このベースラインは **v0.8.0 時点の挙動記録** として扱う。`package.json` と生成物 `metadata.generator/version` は引き続き `sokqa-learning-pack-factory` / `0.7.0` を出力するが、これは [v0.8.0 Release Summary](./v0.8.0-release-summary.md) にある通り、v0.8 が仕様・品質管理・比較設計の整理フェーズであり、実行時生成ロジックを更新していないためである。

## 2. Method

### 2.1 Factory evaluation

以下のコマンドで生成した。

```bash
npm run generate -- --config <config-file>
```

評価は以下を併用した。

- `quality-report.json`
  - `score`
  - `duplicateQuestionRate`
  - `duplicateDocumentRate`
  - `genericDistractors`
  - `genericExplanations`
- manual review
  - Document: 内容の具体性 / テーマ適合性 / 重複率 / 多様性 / 実用性
  - Quiz: 問題の独自性 / 誤答肢品質 / 解説品質 / 正答位置分散 / テーマ適合性
  - TTS: 読みやすさ / 不要記号 / 多言語対応

### 2.2 Studio comparison

Studio 再生成は行っていない。比較は既存の [v0.8 Studio Gap Analysis](./v0.8-studio-gap-analysis.md) を根拠にし、今回の Factory 再生成結果と突き合わせて整理した。

## 4. Overall Result

### 4.1 Conclusion

- `mode: none` は代表 5 Config すべてで **実用不可**。
- `mode: none` の主因は **コンテンツ生成ではなくテンプレート埋め込み**。
- `source_plus` は document は読めるが quiz が全重複で、教材としては不十分。
- `source_only` は source 制約を守るが、quiz が「参照行の再掲」に近く学習効果が低い。
- `exact_text_document` は現行仕様どおり最も安定し、参照素材保持用途では有効。

### 5.2 Cross-cutting findings

#### Document

- 内容の具体性: `none` では全テーマで低い。テーマ文字列差し替えのみ。
- テーマ適合性: 表層的には一致するが、内容実体がない。
- 重複率: 複数 document 構成はすべて 100% 重複。
- 多様性: document role 分割がなく、学習順序も固定スロット。
- 実用性: `none` は学習資料として使えない。

#### Quiz

- 問題の独自性: 同一 quiz 内、複数 quiz 間ともにほぼゼロ。
- 誤答肢品質: `none` では `Skip the basics` / `Apply without checking context` / `Use unclear wording` に収束。
- 解説品質: テーマ再掲が中心で、正答理由・誤答理由の教育価値がない。
- 正答位置分散: 代表 5 Config と追加 `none` / `source_plus` は全問 `answerIndex = 0`。
- テーマ適合性: `source_only` / `exact_text_document` を除くと低い。

#### TTS

- 読みやすさ: `none` の ja pack は英語テンプレ読み上げが混ざり不自然。
- 不要記号: `、,` が継続発生。
- 多言語対応: 英会話 pack でも inline language switch がない。
- 音声最適化: `audioOptimization` は意味単位短縮でなくラベル切り出しになっている。

### 5.3 Additional mode results

| Mode | Strength | Weakness | Baseline verdict |
|---|---|---|---|
| `source_plus` | 参照文書を元に document は読める。要約と tip 追加もある。 | quiz 5問が完全重複。参照ベースでも quiz 品質は壊れている。 | document only なら暫定可 |
| `source_only` | 参照内容保持は安定。document 重複なし。構造的には素直。 | quiz は参照行の再掲で、解説 3件が generic 判定。 | 参照保持用途では可 |
| `exact_text_document` | 仕様どおり quiz 無効。参照テキスト保持品質は高い。 | 教材化ではなく変換用途に限定。 | 現行で最も安定 |
| `none` | 構造だけは常に生成できる。 | 内容品質は空洞。多様性ゼロ。 | 実運用不可 |

## 6. Studio Comparison

### 6.1 Gap summary

既存の [v0.8 Studio Gap Analysis](./v0.8-studio-gap-analysis.md) と今回の再生成結果を合わせると、差分は固定している。

| Area | Studio | Factory baseline |
|---|---|---|
| Document density | 20前後の密度ある学習項目、章タイトルあり | 7-12個の固定テンプレ項目 |
| Document learning arc | 概念の順序立て、例、誤解、まとめが実体を持つ | `Intro -> Theme -> Goal -> Key Points -> Example -> Tip` の固定骨組み |
| Quiz variety | 同一テーマでも問い方・誤答肢・解説が変わる | 同一問題の並び替えのみ |
| Distractors | 同一ドメイン内のもっともらしい誤答 | メタ的な悪い行動テンプレ |
| Explanations | 正答理由と誤答理由を教える | テーマ再掲 |
| TTS | inline 多言語、略語展開、自然な句読点 | 単一言語 prefix、`、,`、ラベル読上げ |

### 6.2 What improved relative to Studio comparison phase

- 改善なし。今回の再生成結果は、既存の v0.8 Studio gap analysis と同じ問題をそのまま再確認した。
- `mode: none` の中核問題は未解消。
- TTS の中核問題も未解消。

## 7. Improvement Candidates

### 7.1 Priority A

次イテレーションで着手する内容は以下のみとする。

1. `none` 向け content-plan 導入
   - document role、sub-topic、quiz viewpoint を先に決める
2. quiz 重複防止の強制
   - 問いの観点重複禁止
   - choice set 重複禁止
   - `answerIndex` 分散必須
3. テーマ適合の誤答肢・解説生成
   - ドメイン内誤答肢
   - 正答理由 + 誤答理由
4. TTS 正規化
   - `、,` 除去
   - ラベル-only TTS 廃止
   - 英会話 pack の inline multilingual 対応

### 7.2 Priority B

1. `source_plus` / `source_only` の quiz 品質改善
   - 参照テキスト依存でも重複しない quiz 生成
2. document 学習順序の明確化
   - 導入、基礎、構造、応用、実例、誤解、まとめの実体化
3. metadata 改善
   - 章タイトル
   - topic-specific tags

### 7.3 Priority C

1. selective TTS
2. Studio parity の追加項目
   - 略語読み
   - author / asset 系の扱い整理
3. preset / config ergonomics

## 8. v0.9.1 Implementation Candidates

v0.9.1 の実装候補は Priority A と一致させる。

| Candidate | Why now | Expected impact |
|---|---|---|
| content-plan generation step | 重複と空洞文書の根本原因 | document / quiz の多様性回復 |
| question blueprint + answerIndex distribution | 全問同型・全問 index 0 を止める | quiz 実用化 |
| domain distractor / explanation generation | generic distractor を止める | quiz 品質回復 |
| TTS normalization + multilingual rule | 英会話 pack と ja TTS の破綻を止める | 音声品質回復 |

## 9. Final Baseline Decision

- v0.8.0 ベースラインはこの文書で固定する。
- 改善対象の中心は `mode: none` の content generation quality。
- 次イテレーションの Priority A は 4件に限定する。
- v0.9.1 では Priority A 以外へ着手しない。
