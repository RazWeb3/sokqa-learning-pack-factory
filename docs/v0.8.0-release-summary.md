# v0.8.0 Release Summary

This document summarizes the v0.8 release preparation phase and closes the
v0.8 documentation cycle before moving to v0.9 quality improvement work.

Scope:

- Documentation and specification release summary only
- No code changes
- No runtime behavior changes

---

## Release Theme

v0.8 formalizes the Factory as a local-first specification, validation, quality
management, and ZIP export tool for SokQA-compatible learning packs.

The current direction is:

- AI IDEs such as Trae, Codex, and ZCode can own generation work.
- The Factory provides generation specifications, JSON rules, validation, quality
  reporting, and local export.
- The Factory does not embed an LLM API.
- The Factory does not adopt a content-engine approach.
- The Factory does not accumulate fixed educational datasets.

---

## Completed in v0.8

### Generation Profile

The v0.6-v0.7 profile system is now treated as part of the generation contract.
The current documentation and Skills reference:

- `targetUser`
- `difficulty`
- `learningStyle`
- `outputMode`
- `tone`
- `detailLevel`
- `exampleLevel`
- `audioOptimization`
- `quizStyle`
- `distractorSource`
- `explanationDepth`
- `practicalExamples`
- `sentenceLength`

### Quality Validation

The Factory now documents quality validation as a first-class output review step.
Validation is no longer only shape and file consistency; content quality signals
are also surfaced through quality metrics.

### Quality Report

`quality-report.json` is written beside `learning-pack.zip` and is not included
inside the ZIP.

It records:

- `score`
- `duplicateQuestionRate`
- `duplicateDocumentRate`
- `genericDistractors`
- `genericExplanations`
- `warnings`

### Studio Gap Analysis

The v0.8 Studio comparison clarified the quality gap between Studio outputs and
Factory outputs:

- Studio documents have stronger learning arcs.
- Studio quizzes have better question diversity.
- Studio distractors are more plausible and theme-specific.
- Studio TTS handles multilingual and pronunciation cases better.

### Automatic Regeneration Design

The v0.8.2 design defines how future quality retry should work:

- Default retry count: 1
- Hard maximum retry count: 2
- No infinite retry loops
- Retry target can be document, quiz, or full pack
- Low-quality final output should still be preserved with retry metadata

### Generation Specification

The v0.8.4 specification defines AI-facing generation requirements for:

- Document structure and learning order
- Quiz viewpoint diversity and distractor quality
- Metadata and `metadata.profile`
- TTS shape and language-aware spoken text
- Reference modes
- Validation and regeneration triggers

### Skills Audit

The v0.8.5 audit confirmed there were no Critical mismatches in Trae Skills and
identified Major gaps around:

- AI IDE ownership wording
- Retry limit consistency
- Quality Report coverage
- Profile coverage
- Generation Specification coverage

### Skills Synchronization

The v0.8.6 synchronization updates aligned Trae Skills with the current
specification:

- AI IDEs are described as generation owners.
- Factory-embedded LLM execution is not assumed.
- Retry limits match v0.8.2.
- `quality-report.json` and quality metrics are reflected.
- `metadata.profile` and profile fields are reflected.
- v0.8.4 generation rules are reflected.

## v0.9 Direction

v0.9 should focus on Generation Quality Improvement:

- Generation Quality Iteration
- Prompt / Generation Rule Refinement
- AI IDE Output Quality Improvements
- Automatic Quality Retry implementation
- Advanced Validation
- Better Document Diversity
- Better Quiz Diversity

v0.9 should build on the v0.8 specification work rather than changing the
Factory into an LLM API integration layer.
